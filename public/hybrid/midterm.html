웹 표준

기기나 브라우저의 종류에 상관없이 누구나 동일한 서비스를 받기 위해 반드시 필요

HTML5의 특징

  • 플랫폼화: 단순한 언어를 넘어 웹 애플리케이션 개발을 위한 플랫폼 기술로 확장
  • 자체 기능 강화: 플러그인(ActiveX 등) 없이도 비디오, 오디오 등 미디어를 재생하고 캔버스(Canvas) 등을 통해 그래픽 구현이 가능
  • 시맨틱 웹: 태그에 의미를 부여하여 구조적인 문서 작성을 지원

웹 브라우저

단순한 번역기(interpreter) 이상의 역할을 수행

브라우저 엔진의 처리 과정

  1. DOM 트리 구성: HTML을 분석하여 메모리에 트리 구조를 만듬
  2. 렌더 트리 구성: CSS 스타일을 적용하여 실제 화면에 보일 엘리먼트를 선별
  3. 레이아웃 구성: 각 요소의 위치와 크기를 계산하여 배치
  4. 페인팅: 브라우저 창에 실제 내용을 그림
MVC 구조: HTML(Model), CSS(View), JavaScript(Controller)가 상호작용하는 구조

모바일 앱의 개발 방식

  • 모바일 웹: 스마트폰 브라우저로 접속하는 웹 페이지
  • 모바일 앱: 기기에 설치되어 실행되는 응용 프로그램
  • 하이브리드 앱: 표준 웹 기술로 웹앱을 개발하고 이를 다시 네이티브앱으로 변환한 다음 배포하는 방식

HTML5 문서의 구조

  • <!DOCTYPE html>: 문서 형식을 HTML5로 지정
  • <html>: 전체 문서의 시작과 끝을 나타내며, lang="ko" 속성으로 주 언어 설정
  • <head>: 문서의 제목, 인코딩 설정, 스타일 및 스크립트 정보
  • <body>: 브라우저 화면에 실제로 나타나는 모든 콘텐츠

  • 브라우저

테이블 태그

  • <table>: 표를 생성

  • <tbody>: 표 꼬리 표시 행들을 그룹화
  • <caption>: 표 제목을 표시

링크 태그, 미디어 태그

웹의 핵심인 연결성과 시각적 요소 담당
  • <a>: href 속성을 통해 다른 문서나 웹사이트로 연결. target="_blank" 속성으로 새 창에서 열기를 지정 가능
  • <img>: 이미지 삽입. src(경로)와 alt(대체 텍스트) 속성 필수
  • <audio>: controls 속성은 화면에 player를 표시하는 역할
  • <video>: autoplay 속성으로 비디오 자동 재생
  • <source>: <audio>, <video>태그의 하위 태그로 사용

시맨틱 태그

문서의 각 부분이 어떤 역할을 하는지 의미를 부여하여 검색 엔진 최적화(SEO)와 접근성을 높임
  • <header>: 상단 영역(로고, 메뉴 등)
  • <nav>: 내비게이션 링크 모음
  • <section>: 주제별 콘텐츠 영역
  • <article>: 독립적인 개별 콘텐츠(뉴스 기사, 블로그 포스트)
  • <aside>: 주요 내용 외의 사이드바 정보
  • <footer>: 하단 영역(저작권, 연락처 등)

CSS

HTML 문서에 적용하는 표준 스타일시트 언어로, 문서의 구조(HTML)와 표현(CSS)을 분리하여 관리할 수 있게 함
  • 확장성: 표현 기능을 다양하게 확장하거나 변경하기 용이
  • 편의성: 레이아웃 구성을 훨씬 간편하게 할 수 있음
  • 재사용성: 독립된 모듈로 작성하여 여러 HTML 문서에 공통으로 적용 가능
  • 생산성: 업무 분담을 통한 전문화와 협업이 가능해져 생산성 향상

스타일 선언 형식

형식: 선택자 { 속성이름 : 속성값 ; }
  • 선택자: 스타일을 적용할 대상(태그, ID, 클래스 등)을 지정
  • 속성 블록: 중괄호 { } 안에 색상, 크기 등 구체적인 스타일을 명세

선택자 유형

  • 전체 선택자 (*): 문서 내 모든 태그 선택
  • 태그 선택자 (태그명): 특정 태그(예: p, h1)를 모두 선택
  • 아이디 선택자 (#): 고유한 ID 값을 가진 요소 하나만 선택
  • 클래스 선택자 (.): 특정 클래스 그룹에 속한 여러 요소 선택
  • 자식 선택자 (>): 부모 바로 아래의 하위 태그만 선택
  • 자손 선택자 (공백): 부모 안의 모든 하위 태그 포함
  • 형제 선택자 (+, ~): 인접한 형제(+)나 뒤에 오는 모든 형제(~) 선택
  • 상태 선택자: 조건을 나타내는 예약어를 명세 (선택자:link, 선택자:visited, 선택자:hover 등)

스타일 적용 우선순위

  1. 선언 방식 기준: 인라인 방식 > 내부 방식 > 외부 방식 > 임포트 방식
  2. 선택자 기준: 아이디 선택자 > 클래스 선택자 > 계층 선택자 > 태그 선택자
  3. 기타 규칙: 하위 엘리먼트에서 직접 정의한 스타일이 상속받은 스타일보다 우선

주요 스타일 속성

  • 글자: font-family(글꼴), font-size(크기), font-weight(굵기), text-shadow(그림자) 등
  • 색상: color(글자색), background-color(배경색), background-image(배경 이미지) 등
  • 단위: 상대 단위(%, em, px 등), 절대 단위(cm, mm, pt 등)
  • 목록: list-style-type으로 불렛 기호 설정, list-style-image로 이미지 사용
  • 테이블: border-collapse(테이블 선 합치기), caption-side(제목 위치) 등

박스 모델(box model)

모든 엘리먼트는 사각형 모양의 공간을 차지하며, 이를 박스 모델이라 함
  • 구조: 내용(Content) → 안쪽 여백(Padding) → 경계선(Border) → 바깥 여백(Margin) 순으로 구성
  • 여백 지정 순서: 위(Top) → 오른쪽(Right) → 아래(Bottom) → 왼쪽(Left) 시계 방향 순서

레이아웃과 화면 배치

컨테이너: 자식 엘리먼트를 포함하는 부모 엘리먼트로, 자식들의 위치 기준이 됨

Position:

  • static: 표준 방식의 정적 위치
  • absolute: 부모 컨테이너를 기준으로 절대 위치 지정
  • fixed: 브라우저 창을 기준으로 고정된 위치에 배치
  • relative: 원래 있어야 할 위치를 기준으로 상대적 이동

Float: 박스를 화면에 띄워 좌우로 정렬하거나 이미지와 텍스트를 함께 배치할 때 유용

뷰포트(viewport)

화면 요소가 보여지는 영역
  • 레이아웃 뷰포트: 장치의 전체 화면 해상도에 해당하는 영역. 실제 브라우저 창 크기와는 무관
  • 비주얼 뷰포트: 전체 페이지 중 현재 화면에 실제로 보이는 영역
모바일 브라우저는 최적화되지 않은 페이지도 모두 보여주기 위해 기본 뷰포트를 실제 화면보다 크게(보통 980px) 설정
👉 글자가 매우 작게 보이는 문제가 발생

뷰포트 메타 태그

<meta> 태그를 사용하여 브라우저에 뷰포트 정보를 제공함으로써 작은 글씨 문제 해결
  • width: 뷰포트 너비 (예: device-width는 장치의 실제 너비)
  • initial-scale: 초기 화면 확대 비율 (기본값 1.0)
  • user-scalable: 사용자의 확대/축소 가능 여부 (yes/no)

미디어 유형

미디어 종류에 따라 각기 다른 스타일시트를 적용
  • all: 모든 장치 (기본값)
  • screen: 컴퓨터, 태블릿, 스마트폰 화면
  • print: 인쇄 장치

@media 규칙을 이용하는 방법

<style>
    @media screen {
        * { font-family: serif; font-size: 24px; } }
    @media print {
        * { font-family: sans-serif; font-size: 8px; } }
</style>

<link> 태그를 이용하는 방법

<head>
    <meta charset="utf-8">
    <title>형식2</title>
    <link rel="stylesheet" href="screen.css" media="screen"/>
    <link rel="stylesheet" href="print.css" media="print"/>
</head>

@import 규칙을 이용하는 방법

<style>
    @import url(screen.css) screen;
    @import url(print.css) print;
</style>

<style> 태그를 이용하는 방법

<style media="screen">
    * { font-family: serif; font-size: 24px; }
</style>
<style media="print">
    * { font-family: sans-serif; font-size: 8px; }
</style>

미디어 쿼리

장치의 해상도, 너비, 회전 상태 등 보다 상세한 조건에 따라 스타일을 적용

기본 구조: @media [only 또는 not] 미디어타입 and (조건) { ... }

주요 속성

  • width, height: 웹 페이지의 가로 너비 및 세로 높이
  • device-width, device-height: 장치의 물리적 해상도 너비 및 높이
  • orientation: 장치의 회전 상태 (portrait 세로, landscape 가로)

  • HTML
  • 브라우저
<style>
    @media only screen and (orientation: portrait) {
        div { background-color: lightyellow; color: black; }
    }
    @media only screen and (orientation: landscape) {
        div { background-color: purple; color: white; }
    }
</style>

모바일 웹앱과 프레임워크

  • 모바일 웹앱: 네이티브 앱 같은 모바일 웹 페이지
  • 다양한 모바일 장치로의 배포가 쉽고 네이티브 앱과 유사한 인터페이스 제공
  • 사용자 경험(UX) 강조: 작은 화면에서 효율적인 상호 작용을 위해 UX가 매우 중요

제이쿼리 모바일 프레임워크

  • 웹 브라우저 안에서 동작하는 웹앱을 개발하기 위한 통합 UI 프레임워크
  • 연동 방식: HTML5 문서 내에 URL을 추가하는 CDN 방식, 파일을 직접 내려받는 다운로드 방식
  • CSS3 파일, 제이쿼리 스크립트 파일, 제이쿼리 모바일 스크립트 파일 순으로 HTML5 마크업에 포함되어야 함

제이쿼리 모바일 기본 구조

제이쿼리 모바일의 페이지는 하나의 화면 단위를 의미, data-role 속성을 사용하여 구조를 정의
  • page: 전체 화면 영역을 감싸는 컨테이너
  • header: 페이지 상단의 툴바 영역
  • content: 실제 페이지 내용이 표시되는 중앙 영역
  • footer: 페이지 하단의 툴바 영역

화면 구성

  • 단일 페이지 템플릿: 하나의 HTML 문서에 하나의 페이지만 정의
  • 다중 페이지 템플릿: 하나의 HTML 문서에 여러 페이지를 정의. 첫 번째 페이지만 먼저 보이고 나머지는 링크를 통해 Ajax 방식으로 전환

페이지 연결 방식

내부 페이지 연결

  • 하나의 HTML5 파일 안에 여러 페이지를 명세하여 서로 연결
  • 다중 페이지 템플릿을 적용하는 형태
  • 각 내부 페이지들은 Ajax를 통해 내부 링크 방식으로 연결됨

외부 페이지의 내부 링크 방식

  • DOM 트리를 새롭게 생성하지 않고 대신 연결된 외부 페이지를 현재의 DOM 트리에 추가
  • DOM 트리에서 중복되는 엘리먼트 노드는 제거됨(head, body)
  • Ajax에 의해 자연스러운 화면 전환 애니메이션 적용 가능

외부 페이지의 외부 링크 방식

  • Ajax를 사용하지 않는 고전적 방식
  • 기존 DOM 트리를 제거하고 새로운 DOM 트리를 구성
  • 기존 코드와의 충돌을 차단할 때 유용함
  • data-ajax="false", rel=”external”, target=”_blank” 사용

페이지 전환 효과

  • CSS3를 사용, 페이지 이동 과정에서 화면 전환 효과를 제공
  • <a> 링크 태그 안에 data-transition 속성값을 설정
  • Data-direction="reverse": 역방향이나 반대 애니메이션
  • 내부링크 방식에만 애니메이션 적용됨
  • slide (왼쪽으로 수평이동. 기본 효과)
  • slideup (위쪽으로 수직이동)
  • slidedown (아래쪽으로 수직이동)
  • pop (페이지 확장)
  • fade (페이드인)
  • flip (페이지 회전)
  • turn (페이지 뒤집기)
  • flow (페이지 축소 후 좌우이동)

What The Course Is About

Native App vs. Web App vs. Hybrid App

Hybrid Application is a software application that combines elements of both native apps and web applications. Hybrid apps are essentially web apps that have been put in a native app shell.

Hybrid apps are popular because they allow developers to write code for a mobile app once and still accommodate multiple platforms.

Because hybrid apps add an extra layer between the source code and the target platform, they may perform slightly slower than native or web versions of the same app.

  • 연결 리스트뷰: 기본 리스트뷰의 각 항목에 링크 정보를 추가
  • data-inset="true": 콘텐츠 영역 안에 내부 블록을 형성하고 그 안에 리스트뷰를 표시
  • 분할 리스트뷰: 각 항목 영역을 분리하여 둘 이상의 링크를 통해 여러 페이지와의 연동
  • 카운트 버블: 리스트뷰 항목의 둥근 거품 방울 모양의 화면 요소
  • HTML
  • 브라우저
<ul data-role="listview" data-inset="true">
    <li><a href="#">리스트1</a></li>
    <li><a href="#">리스트2</a><a href="#">상세내용</a></li>
    <li><a href="#">리스트3<span class="ui-li-count">5</span></a></li>
</ul>
리스트뷰
  • 리스트1
  • 리스트2상세내용
  • 리스트35

리스트 구분자

  • 원하는 항목을 찾기 쉽도록 각 그룹별로 분류명을 표시
  • <li> 태그 안에 data-role="listdivider" 속성을 추가
  • 항목들이 하나의 그룹으로만 분류되는 경우, 리스트 구분자는 항목들의 제목 역할을 함

멀티미디어 적용

  • 썸네일 리스트뷰: 리스트뷰의 각 항목 내용으로 <img> 태그를 추가
  • 아이콘 리스트뷰: <img> 태그에 ui-li-icon 클래스 속성을 추가
  • 여러 줄 리스트뷰: 표시 내용의 중요도에 따라 강조할 내용은 <h> 제목 태그로, 나머지 내용은 <p> 문단 태그를 사용하여 일정한 형식으로 상세한 설명과 관련 정보들을 여러 줄에 표시

검색 필터 리스트

data-filter="true" 속성을 추가하면 상단에 키워드 검색창이 생성되어 항목을 필터링할 수 있음
  • HTML
  • 브라우저
<ul data-role="listview" data-inset="true" data-filter="true">
    <li data-role="listdivider">ㄱ</li>
    <li><a href="#">강</a></li>
    <li data-role="listdivider">ㅂ</li>
    <li><a href="#">바다</a></li>
    <li><a href="#">바다새</a></li>
</ul>
리스트뷰
  • ㄱ
  • 강
  • ㅂ
  • 바다
  • 바다새

페이지 탭

  • data-role="tabs"를 사용
  • 한 화면 내에서 여러 탭 페이지를 전환하며 브라우징
<div data-role="tabs">
    <div data-role="navbar">
        <ul>
            <li><a href="#tab1">탭1</a></li>
            <li><a href="#tab2">탭2</a></li>
        </ul>
    </div>
    <div id="tab1">
        첫번째 탭 내용
    </div>
    <div id="tab2">
        두번째 탭 내용
    </div>
</div>

컬럼 토글 테이블

  • 화면 폭에 따라 테이블 컬럼을 자동으로 숨기거나 표시
  • data-priority 속성으로 컬럼별 우선순위를 지정

숨김 패널

  • 메뉴나 폼 등을 숨겨두었다가 필요할 때 페이지 전환 없이 끌어내는 기능
  • data-role="panel"로 정의
  • overlay 방식으로 페이지 위에 펼칠 수 있음
